1. สรุปสถานะปัจจุบัน

ทำไว้แล้ว

  • ผู้ถูกชวนได้รับ VIP 3 วัน เมื่อเข้าระบบครั้งแรกผ่าน invite
  • ผู้ชวนได้รับ VIP 3 วัน + บัตรค่าประสบการณ์ x2 1 ใบ เมื่อเพื่อนเข้ามาใช้งาน Application สำเร็จ
  • เพิ่มช่องทางการได้รับ VIP จาก Redeem Code
  • VIP ยังซื้อได้ผ่าน Subscription ราคา 199 บาท / เดือน
  • VIP ยังได้รับจาก Level Reward ตามกฎหลักของระบบ

ยังต้องคุมให้ชัด

  • ต้องนิยามคำว่า “เข้ามาใช้งาน Application สำเร็จ” ให้ Server ตรวจได้
  • ต้องมี Anti-fraud rule แบบละเอียด
  • ต้องมี event lifecycle ของ invite
  • ต้องมี data model / server validation สำหรับ referral
Decision updated: คนชวนจะได้รางวัลเป็น VIP 3 วัน + บัตรค่าประสบการณ์ x2 1 ใบ เมื่อเพื่อนถูกชวนและเข้ามาใช้งาน Application สำเร็จ ไม่ใช่แค่กด link หรือสมัครทิ้งไว้

2. Product Goal

ระบบชวนเพื่อนมีเป้าหมายเพื่อเพิ่มผู้ใช้ใหม่ให้ PIP ผ่านการแนะนำจากผู้ใช้เดิม โดยให้รางวัลพอรู้สึกคุ้ม แต่ไม่แรงจนเกิดการฟาร์มบัญชีปลอม

Core Principle: Invite reward ต้องช่วยให้ผู้ใช้ใหม่ทดลองระบบได้ง่ายขึ้น และให้ผู้ชวนรู้สึกมีแรงจูงใจ แต่ทุกอย่างต้องตรวจสอบได้จาก Server

3. Final Reward Logic

Role Reward Condition Status
ผู้ถูกชวน VIP 3 วัน สมัครใหม่ + Login ครั้งแรกผ่าน Invite Code สำเร็จ Approved
ผู้ชวน VIP 3 วัน + บัตรค่าประสบการณ์ x2 1 ใบ เพื่อนสมัครใหม่ + Login ครั้งแรก + เริ่มใช้งาน Application สำเร็จ Approved
Final rule: ผู้ชวนได้รับ VIP 3 วัน + บัตรค่าประสบการณ์ x2 1 ใบ เมื่อเพื่อนสมัครใหม่ผ่าน invite และเริ่มใช้งาน Application จริง โดยการให้รางวัลทั้งหมดต้องทำจาก Server เท่านั้น

4. Invite Flow

1 Generate Code ผู้ใช้เดิมได้รับ invite code หรือ invite link ของตัวเอง
2 Share Invite แชร์ให้เพื่อนผ่านช่องทางใดก็ได้ เช่น LINE, Messenger, Copy Link
3 New User Signup ผู้ถูกชวนต้องเป็นบัญชีใหม่เท่านั้น
4 Invitee Reward ผู้ถูกชวนได้รับ VIP 3 วันหลัง Login ครั้งแรกสำเร็จ
5 Referrer Reward ผู้ชวนได้รับ VIP 3 วัน + x2 EXP Ticket 1 ใบหลังเพื่อนเริ่มใช้งาน Application สำเร็จ

5. Definition: เริ่มใช้งาน Application สำเร็จ

เพื่อกันการสมัครทิ้งไว้เพื่อฟาร์มรางวัล คำว่า “เริ่มใช้งาน Application สำเร็จ” ต้องหมายถึงผู้ถูกชวนผ่านเงื่อนไขขั้นต่ำที่ Server ตรวจสอบได้

Recommended trigger: ให้ถือว่าเริ่มใช้งานสำเร็จเมื่อผู้ถูกชวน Login ครั้งแรก + เข้าใช้งานหน้า Practice หรือ Mini Game อย่างน้อย 1 ครั้ง หรือทำ Onboarding สำเร็จตาม flow หลักของแอป
  • ไม่ควรแจก reward ผู้ชวนจากการสมัครบัญชีอย่างเดียว
  • ไม่ควรแจก reward ผู้ชวนจากการเปิด invite link อย่างเดียว
  • Server ต้องเป็นคนยืนยัน usage event เท่านั้น
  • ถ้าต้องการเข้มขึ้นในอนาคต สามารถเปลี่ยน trigger เป็น “ผ่าน Mini Game ด่านแรก” ได้

6. Eligibility Rules

ผู้ถูกชวน

  • ต้องเป็นบัญชีใหม่
  • ต้องไม่เคยรับ invite reward มาก่อน
  • ต้อง Login ครั้งแรกผ่าน invite code/link
  • ถ้า redeem code หรือ invite code ซ้อนกัน ให้ใช้ source ที่เกิดก่อนและ valid ก่อน

ผู้ชวน

  • ต้องเป็นบัญชีปกติที่ยังไม่ถูกแบน
  • ต้องไม่สามารถชวนบัญชีตัวเองได้
  • ได้รับ reward หลังผู้ถูกชวนสมัครใหม่, Login ครั้งแรก, และเริ่มใช้งาน Application สำเร็จเท่านั้น
  • ช่วงแรกจำกัด reward สูงสุด 10 ครั้ง / เดือน
Must not: ห้ามแจก reward ผู้ชวนทันทีหลังกรอก invite code เฉย ๆ เพราะจะโดนสร้าง account ปลอมเพื่อฟาร์มรางวัลง่ายมาก ต้องรอให้เพื่อนเริ่มใช้งาน Application จริงก่อน

7. Anti-fraud Rules

Rule Description Action
One account rule บัญชีหนึ่งรับ invite reward ได้ครั้งเดียว Block duplicate reward
Self invite prevention ผู้ชวนและผู้ถูกชวนต้องไม่ใช่บัญชีเดียวกัน Reject invite
Device / identity signal ตรวจซ้ำจาก device id, email, phone, auth provider หรือ signal อื่นที่ระบบมี Flag for review or reject
Monthly cap ผู้ชวนรับ reward ได้สูงสุด 10 ครั้ง / เดือนในช่วงแรก Hold reward after cap
Reward delay รางวัลผู้ชวนออกหลังเพื่อนเริ่มใช้งาน Application สำเร็จ Prevent signup farming
Server authoritative การให้ VIP, Ticket, invite status ต้องคำนวณและบันทึกจาก server เท่านั้น Client cannot grant reward

10. Server Rules

  • Client สามารถส่ง invite code ได้ แต่ไม่มีสิทธิ์ grant reward เอง
  • Server ต้องตรวจว่า invite code valid, active, และผูกกับ referrer จริง
  • Server ต้องตรวจว่า invitee เป็นบัญชีใหม่ และไม่เคยรับ invite reward มาก่อน
  • Server ต้องตรวจว่า invitee เริ่มใช้งาน Application สำเร็จก่อนให้ reward ผู้ชวน
  • Server ต้องบันทึก reward ลง RewardLedger ทุกครั้ง
  • Server ต้องกัน duplicate transaction ด้วย idempotency key หรือ unique constraint
  • Server ต้องไม่ให้ผู้ใช้แก้ referral status เองจาก client
Implementation note: RewardLedger ควรเป็น source of truth ของรางวัลทั้งหมด ทั้ง VIP, x2 Ticket, Level Reward, Redeem Code และ Invite Reward

Case Expected Behavior

ผู้ชวนไม่มี VIP เพิ่ม VIP 3 วันนับจากเวลาที่ reward granted
ผู้ใช้มี VIP อยู่แล้ว เพิ่มวัน VIP ต่อจากวันหมดอายุเดิม
ผู้ใช้มี Subscription VIP บันทึก reward ได้ และให้ bonus VIP days ต่อท้ายวันหมดอายุเดิมเพื่อเข้าใจง่าย
ผู้ใช้รับ VIP จากหลาย source ในวันเดียวกัน รวมวัน VIP ได้ แต่ต้องมี ledger แยก source ให้ชัด

12. UI / UX Requirements

Invite Entry

  • หน้า Profile หรือ Reward Center มีปุ่มชวนเพื่อน
  • แสดง invite code และปุ่ม copy link
  • ข้อความต้องสั้นและเข้าใจง่าย

Invitee Message

  • บอกชัดว่าได้รับ VIP 3 วัน
  • ห้ามสื่อว่าฟรีตลอดชีพ
  • ต้องบอกว่า reward เกิดหลัง login ครั้งแรก

Referrer Message

  • บอกเงื่อนไขว่าเพื่อนต้องเริ่มใช้งาน Application สำเร็จก่อน
  • แสดง progress เช่น รอเพื่อนเริ่มใช้งาน
  • แสดง reward cap ถ้ามี
Suggested Thai UI wording: “ชวนเพื่อนรับ VIP 3 วัน เพื่อนเริ่มใช้งานครั้งแรกสำเร็จ คุณรับ VIP 3 วัน + x2 EXP Ticket 1 ใบ”

13. Edge Cases

Scenario Rule
Invite code หมดอายุ ไม่ให้ reward และแจ้งให้ผู้ใช้ขอ link ใหม่
ผู้ถูกชวนสมัครแล้ว แต่ยังไม่เริ่มใช้งาน Application จริง ผู้ถูกชวนได้ VIP 3 วันแล้ว แต่ผู้ชวนยังไม่ได้ VIP 3 วัน + x2 EXP Ticket
ผู้ถูกชวนลบบัญชีแล้วสมัครใหม่ Server ต้องตรวจ duplicate identity signal และไม่ควรให้ reward ซ้ำ
Invite code หลายอันในเครื่องเดียว ใช้ code ที่ถูก claim สำเร็จครั้งแรกเท่านั้น
ผู้ชวนถึง cap รายเดือนแล้ว ไม่แจก reward เพิ่ม หรือ hold ไว้ตาม policy ที่เลือก

16. Final Decision Summary

Approved Rule

ผู้ถูกชวนได้รับ VIP 3 วัน เมื่อสมัครใหม่และ Login ครั้งแรกผ่าน invite สำเร็จ ส่วนผู้ชวนได้รับ VIP 3 วัน + บัตรค่าประสบการณ์ x2 1 ใบ เมื่อผู้ถูกชวนสมัครใหม่และเริ่มใช้งาน Application สำเร็จ

Why this is balanced

กฎนี้ให้แรงจูงใจทั้งสองฝั่งแบบชัดเจน ผู้ถูกชวนได้ VIP เพื่อทดลองใช้ ส่วนผู้ชวนได้ทั้ง VIP และ x2 Ticket แต่ต้องมีเงื่อนไขการใช้งานจริงและ Anti-fraud เพื่อกันการฟาร์มบัญชีปลอม

Do not change without approval

ห้ามเพิ่ม reward ผู้ชวนให้สูงกว่า VIP 3 วัน + x2 EXP Ticket 1 ใบ จนกว่าจะมีข้อมูลจริงจาก analytics หลังเปิดใช้งาน
