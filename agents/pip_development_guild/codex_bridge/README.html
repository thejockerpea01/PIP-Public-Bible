Codex Bridge สำหรับ PIP Agent Dashboard

Codex Bridge เป็นเซิร์ฟเวอร์ Node.js แบบ local-only ที่รับงานจาก agent_dashboard.html แล้วสั่ง codex exec ในเครื่องนี้เท่านั้น

เส้นทางการทำงานคือ Dashboard ส่ง JSON ไปที่ bridge จากนั้น bridge สร้างคำสั่ง Codex เอง จัดคิวงาน เก็บ log และรายงานสถานะกลับไปที่หน้า dashboard

วิธีเริ่มใช้งาน

cd /Users/Pae/Developer/PIP/PIP-Public-Bible/agents/pip_development_guild/codex_bridge
node codex_bridge.js

เมื่อเริ่มสำเร็จ bridge จะฟังที่ http://127.0.0.1:8787

วิธีหยุด

กลับไปที่ terminal ที่รัน bridge แล้วกด Control+C

Endpoint หลัก

  • GET /health ตรวจว่า bridge พร้อมใช้งานหรือไม่
  • POST /tasks ส่งงานแบบ structured payload จาก dashboard
  • GET /tasks ดูรายการงานทั้งหมดในหน่วยความจำของ bridge รอบนี้
  • GET /tasks/:id ดูสถานะงานเดียว

กฎความปลอดภัย

  • ระบบนี้เป็น local-only เท่านั้น
  • ห้าม expose port 8787 ขึ้น internet
  • ห้ามรันด้วย danger-full-access
  • bridge ไม่รับ shell command จาก browser และไม่ให้ browser เปลี่ยน working directory
  • working directory ถูกล็อกไว้ที่ /Users/Pae/Developer/PIP/PIP-Public-Bible
  • ควร commit ก่อนรันงานใหญ่
  • หลัง Codex รันเสร็จ ให้ตรวจ git diff ทุกครั้ง

Logs และ Tasks

ไฟล์ log ของแต่ละงานจะอยู่ใน logs/ และสถานะงานจะถูกบันทึกใน tasks/

bridge รันงานทีละหนึ่งงานเท่านั้น งานใหม่จะเข้าคิวและรอจนกว่างานก่อนหน้าจะจบ

Troubleshooting

  • ถ้า dashboard แสดงว่ายังไม่ได้เปิด Codex Bridge ให้ตรวจว่า terminal ยังรัน node codex_bridge.js อยู่
  • ถ้า task เป็น failed ให้เปิดไฟล์ log ใน logs/
  • ถ้า Codex start ไม่ได้ ให้ตรวจว่า codex CLI อยู่ใน PATH ของ terminal ที่รัน bridge
  • ถ้า port ถูกใช้อยู่ ให้ปิด process เดิมก่อน เพราะ bridge นี้ตั้งใจใช้ port 8787 เท่านั้น
