Practical Usage

Taste Skill

How to use it without letting taste override product truth

Taste Skill is best used as a practical frontend/UI refinement layer. It is appropriate for polish, visual cleanup, and premium interface direction once the product logic, role mapping, and system constraints are already stable.

Best Use Cases

Use it where interface quality matters and the product rules are already known.

Frontend Polish

Use for Frontend/UI Polish

  • Layout cleanup and stronger page composition.
  • Typography hierarchy and spacing discipline.
  • Visual hierarchy, interaction rhythm, and premium UI direction.
  • Subtle motion decisions that support the product rather than distract.

Use for Review and Audit Work

  • Redesign audits when a page looks generic or over-carded.
  • Interface review when dense workflows need clearer scanning behavior.
  • Judging whether a UI feels intentional, domain-appropriate, and restrained.

Guardrails for PIP

Use it selectively, not as a blanket rule.

PIP Guardrails
Do not use it blindly for every product screen.

Some screens need operational clarity more than visual flourish. Taste Skill should support the actual job of the screen, not force a single visual style everywhere.

For PIP, follow current Product Bible and approved design system first.

Product rules, approved UI canon, and any locked role or layout behavior take priority over taste-driven revisions.
