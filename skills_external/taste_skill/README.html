Taste Skill

HTML-first reference for UI polish and anti-generic frontend direction

Taste Skill is an external frontend/UI design reasoning resource from the public Leonxlnx/taste-skill repository. In this repo, it is documented as readable HTML companion files rather than relying on raw markdown skill files alone.

Open Practical Usage Open Source Reference Original GitHub Repo

What It Is

External design reasoning support for frontend polish, visual hierarchy, and premium UI decisions.

UI Reasoning

When to Use It

  • Frontend/UI polish after the product and layout rules are already clear.
  • Typography, spacing, visual hierarchy, motion restraint, and premium surface quality.
  • Redesign audits when a page feels generic, noisy, or visually weak.

When Not to Use It

  • Do not use it as a replacement for product rules or source-of-truth logic.
  • Do not apply it blindly to every screen without checking the current system constraints.
  • Do not let taste-driven changes override approved canon or locked UI behavior.

PIP Relevance

  • Useful for Guild Dashboard, Agent Dashboard, and other interface-heavy HTML work.
  • Helpful when refining documentation hubs, control rooms, or polished app-facing UI.
  • Always subordinate to Product Bible rules and approved design system decisions.

How It Relates to PIP

Use Taste Skill as support, not authority.

Guardrail

For Agent Dashboard and UI Work

Taste Skill can help reduce generic web-layout choices and push the interface toward clearer hierarchy, stronger composition, tighter spacing, and more intentional visual rhythm.

Authority Order

For PIP, follow the current Product Bible, approved role mapping, and active design system first. Use Taste Skill only after those constraints are respected.
